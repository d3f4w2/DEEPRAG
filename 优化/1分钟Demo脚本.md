# 1分钟 Demo 脚本（提问 -> 检索 -> 证据 -> 答案；语音/图片入库后可命中）

## 0. 演示前准备（提前完成）

1. 后端启动：`uvicorn backend.main:app --host 0.0.0.0 --port 8000`
2. 前端启动：`cd frontend && npm run dev`
3. 确认页面可访问：`http://localhost:5173`

## 1. 时间轴脚本（60秒）

### 0s - 15s：展示证据驱动问答

在 Chat 页面输入：

```text
除了 AMOLED 和 OLED 还有哪些显示类型？
```

讲解词：
- “这里不是直接生成答案，它会先走路径检索，再走片段检索。”
- “最终答案带 `### 证据`，每条证据都能追溯来源文件。”

### 15s - 30s：展示工具链路与可解释性

讲解词：
- “你能看到工具调用过程：`search_paths -> retrieve_sections`。”
- “证据池通过后才放行最终回答，降低无证据幻觉。”

### 30s - 45s：展示语音入库

切到“语音入库”页，快速操作：
1. 点击“录音开/关”说一句短语（或用已有文本）
2. 点击“同步AI草稿”
3. 点击“确认入库”

讲解词：
- “语音先生成草稿，再人工确认，最后写入知识库并同步索引。”

### 45s - 60s：回到 Chat 验证“刚入库可检索”

输入（可按实际作者名替换）：

```text
刚才这条语音记录的核心内容是什么？
```

讲解词：
- “同一套检索链路可以直接命中刚才入库的语音笔记，并给出证据引用。”

## 2. 备选问题（防止主问题波动）

1. `SW-2100-Flagship 是什么定位？请给证据。`
2. `Southwest Region 有多少门店？请给证据。`
3. `图片入库后，这张图里的关键信息是什么？请给证据。`

## 3. 收尾一句话（面试版）

“这个项目的核心不是‘回答出来’，而是‘有证据地回答出来’，并且能把语音/图片等新信息快速纳入同一检索闭环。”

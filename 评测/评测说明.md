# 小评测方案（面试友好版）

这套评测是按你当前项目量身定制的，目标是：

- 题量控制在 `30-50`（当前为 `40` 题）
- 题目尽量“单事实”，便于快速看出优化收益
- 固定跟踪 4 个核心指标：
  - `准确率`
  - `证据命中率`
  - `平均 token`
  - `平均延迟`

---

## 文件说明

- 评测集：[小评测集_40题.json](./小评测集_40题.json)
- 评测脚本：[运行小评测.py](./运行小评测.py)
- 结果目录：`评测/结果/`（脚本会自动创建）
- 前端测评界面：切到页面顶部 `测评` Tab（异步任务模式）

---

## 评测集设计原则（为什么这样做）

1. 覆盖面够用：手表、耳机、供应商三类知识都覆盖到。  
2. 判分简单：每题都给了`判分关键词组`，不走复杂 LLM judge，结果稳定可复现。  
3. 证据可检验：每题都给了`期望证据文件`，可以算证据命中率。  
4. 面试可讲：能清楚展示“检索策略优化后，token 和延迟如何下降，同时准确率/证据命中不掉”。

---

## 一键运行

先确保后端已启动（默认 `http://127.0.0.1:8000`）。

```bash
python 评测/运行小评测.py --run-name baseline
```

跑完后会在 `评测/结果/` 生成：

- `baseline_汇总.json`
- `baseline_明细.json`
- `baseline_报告.md`

---

## 前端测评工作台（异步）

已新增独立测评页面，支持：

- 选择评测集
- 一键发起后台评测任务
- 实时查看任务进度（评测阶段 + LLM 复盘阶段）
- 自动展示四项核心指标
- 自动定位失败题
- 自动显示 LLM 复盘报告

如果你选了“自动复盘”，后端会在评测结束后读取完整明细并分块送给 LLM 分析，再产出最终复盘建议。

---

## 做“优化前 vs 优化后”对比

优化后再跑一轮，并带上对比文件：

```bash
python 评测/运行小评测.py --run-name optimized --compare 评测/结果/baseline_汇总.json
```

此时 `optimized_报告.md` 里会自动给出对比表。

---

## 常用参数

- `--base-url`：后端地址（默认 `http://127.0.0.1:8000`）
- `--provider`：可选，指定 provider
- `--timeout`：单题超时秒数（默认 `180`）
- `--max-questions`：只跑前 N 题（快速 smoke test 用）
- `--dataset`：自定义评测集路径
- `--output-dir`：自定义结果输出目录

示例（先小跑 5 题）：

```bash
python 评测/运行小评测.py --run-name smoke --max-questions 5
```

---

## 指标口径（建议你面试时直接照这个说）

1. `准确率`  
每题按`判分关键词组`判定，所有关键词组都命中则该题记为准确。

2. `证据命中率`  
答案中引用的“来源文件”与该题期望证据文件有交集，则记命中。

3. `平均 token`  
按流式 `usage` 累加得到单题总 token，再对全题集求平均。

4. `平均延迟`  
单题从请求发出到流结束（`done`）的耗时，单位毫秒，最后求平均。

---

## 如何扩展评测集

在 `小评测集_40题.json` 里按同样格式加题即可，核心字段：

- `问题`
- `判分关键词组`（一个组里是“同义词 OR”；多个组之间是“AND”）
- `期望证据文件`

这套格式的好处是：你以后可以快速加题，不用改脚本。

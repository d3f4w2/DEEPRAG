## 6. 与 baseline 的可比性说明（避免被追问）

- baseline 文件：`评测/结果/baseline_40q_汇总.json`
- baseline 属于旧口径（`准确率/证据命中率`），不含 RAGAS 三指标。
- 因此：RAGAS 三指标不能做严格同口径横比。
- 可直接横比项：token 与延迟
  - baseline 平均 token：`6177.15`
  - baseline 平均延迟：`13721.10 ms`

## 7. 面试建议话术（增强版）

“我把 Deep RAG 做到了 40 题全量闭环可跑通，失败题数 0。核心价值不是单点分数，而是工程可落地：检索链路从整文件召回升级为 `search_paths -> retrieve_sections`，并且答案必须带证据块。单题实测里，上下文体量下降了 59.9%，首轮检索下降 82.1%，同时回答覆盖更完整。最新全量 run 我只对外讲可追溯的 completion、token、latency；RAGAS 在当前 provider 下还有解析兼容性问题，所以我把历史可测值、最新实测值、目标值三者分开展示，避免混淆口径。”

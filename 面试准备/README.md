# Deep RAG 面试准备总入口

这不是一个“资料堆”，而是一套按顺序阅读和练习的面试训练包。

你的目标不是“知道项目做了什么”，而是做到下面四件事：

1. 能在 `10秒 / 30秒 / 90秒 / 3分钟` 四种长度下把项目讲清楚。
2. 能把“你亲自做了什么”说得具体、可信、有技术细节。
3. 能稳稳回答高频技术追问，不乱吹、不心虚、不被追问击穿。
4. 能现场 Demo，即使出现波动，也知道怎么救场。

## 推荐学习方式

不是“默读完”就结束，而是按下面方式：

1. 每读完一份，就合上文档，用自己的话复述一遍。
2. 每份文档里只要出现“请你现在说一遍”，你就必须开口说。
3. 第 1 轮阅读追求“理解”。
4. 第 2 轮阅读追求“记忆”。
5. 第 3 轮阅读追求“脱稿表达”。

## 建议时间安排

如果你有半天时间：

1. 第 1 小时：读 `01~05`
2. 第 2 小时：读 `06~08`
3. 第 3 小时：做 `09`
4. 最后 20 分钟：只看 `10` 和 `11`

如果你只有 90 分钟：

1. 必读 `01 / 04 / 05 / 06 / 07 / 11`
2. 其中 `05` 和 `07` 一定不能跳

## 读完后的达标标准

你读完整个文件夹后，至少应该达到下面水平：

1. 面试官问“你这个项目是什么”，你能在 30 秒内说清楚。
2. 面试官问“你做了哪些优化”，你能说出 `两阶段检索 / 证据放行 / 评测闭环 / 多模态入库 / 稳定性修复`。
3. 面试官问“为什么最新 RAGAS 不好看”，你能诚实解释口径，不慌不乱。
4. 面试官让你现场讲架构，你能按“前端 -> 后端 -> 检索 -> 证据 -> 评测”顺序讲。
5. 面试官追问“这是不是你自己做的”，你能讲出具体改动、问题、取舍和结果。

---

## 先记住一句总原则

这个项目面试时最强的点，不是“分数多高”，而是：

`它是一个能跑通、能解释、能追溯、能继续优化的工程化 RAG 项目。`

只要你把这条主线守住，面试就会很稳。
